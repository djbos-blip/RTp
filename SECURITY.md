# Безопасность

Если нашёл проблему безопасности, не публикуй приватные ключи, комнаты, реальные
подписки или серверные адреса в открытом issue.

## Что считается чувствительными данными

- `keyhex` / 64-hex ключи `olcrtc`;
- приватные комнаты Jitsi, Telemost, wbstream, jazz;
- SOCKS username/password;
- IP-адреса личных серверов;
- полные логи, где есть реальные URI профилей.

## Как сообщать

Создай приватный канал связи с владельцем репозитория или отправь issue без
секретов, описав только тип проблемы и способ воспроизведения на тестовом
профиле.

## Базовые правила проекта

- Секреты не должны попадать в `UserDefaults`, README, screenshots или CI logs.
- Для постоянных секретов используется Keychain.
- Логи должны помогать диагностике, но не печатать полный `olcrtc://` URI.
- GitHub Actions не должен требовать production-секретов для обычной сборки IPA.
