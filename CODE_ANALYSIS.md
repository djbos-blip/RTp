## 🐛 Критические проблемы

### 1. **Race Condition в `LocalProxyController`** ⚠️⚠️⚠️

**Файл:** `LocalProxyController.swift`  
**Строки:** 88-120, 122-160

**Проблема:**
```swift
func start(profile: OlcRTCProfile) async {
    reconnectTask?.cancel()
    reconnectTask = nil
    foregroundCheckTask?.cancel()
    foregroundCheckTask = nil
    stopWatchdog()
    status = .starting
    // ...
}
```

Между отменой задач и началом новой операции может произойти race condition, если пользователь быстро нажимает кнопки.

**Решение:**
Добавить флаг `isOperationInProgress` для блокировки параллельных операций.

---

### 2. **Отсутствие обработки ошибок сети** ⚠️⚠️

**Файл:** `LocalProxyController.swift`  
**Строки:** 195-230

**Проблема:**
```swift
private func handlePathUpdate(_ snapshot: NetworkPathSnapshot) {
    networkName = snapshot.name

    if lastPathSignature == nil {
        lastPathSignature = snapshot.signature
        return
    }
    // Нет обработки быстрых переключений сети
}
```

При быстром переключении между Wi-Fi и LTE (например, в метро) может происходить множество реконнектов.

**Решение:**
Добавить debounce для сетевых изменений (задержка 2-3 секунды).

---

### 3. **Watchdog может конфликтовать с ручным рестартом** ⚠️⚠️

**Файл:** `LocalProxyController.swift`  
**Строки:** 265-295

**Проблема:**
```swift
private func runWatchdogTick() async {
    // ...
    if consecutiveHealthFailures >= 2 {
        scheduleRestart(profile: activeProfile, ...)
    }
}
```

Если пользователь нажимает "Перезапустить" в момент, когда watchdog тоже решает перезапустить, может произойти двойной рестарт.

**Решение:**
Проверять `status` перед рестартом в watchdog.

---

### 4. **Потенциальная утечка памяти в Task** ⚠️

**Файл:** `LocalProxyController.swift`  
**Строки:** 237-248

**Проблема:**
```swift
reconnectTask = Task { [profile] in
    // ...
    await restartNow(profile: profile, reason: reason)
}
```

Захват `profile` может привести к retain cycle, если профиль содержит ссылки.

**Решение:**
Использовать `[weak self]` и проверять `self`.

## ⚠️ Важные проблемы

### 5. **Недостаточный таймаут для SOCKS проверки**

**Файл:** `OlcRTCEngine.swift`  
**Строки:** 56-70

**Проблема:**
```swift
static func checkTunnelConnectivity(
    port: Int,
    credentials: SocksCredentials,
    timeoutNanoseconds: UInt64 = 8_000_000_000  // 8 секунд
) async -> Bool
```

8 секунд может быть недостаточно на медленных сетях (особенно LTE с плохим сигналом).

**Решение:**
Увеличить до 12 секунд или сделать адаптивным.

---

### 6. **Отсутствие экспоненциального backoff**

**Файл:** `LocalProxyController.swift`  
**Строки:** 265-295

**Проблема:**
Watchdog проверяет каждые 45 секунд с фиксированным интервалом. При постоянных проблемах это создаёт лишнюю нагрузку.

**Решение:**
Добавить экспоненциальный backoff: 45s → 90s → 180s → 300s (макс).

---

### 7. **Нет обработки фоновых ограничений iOS**

**Файл:** `BackgroundKeepAlive.swift`  
**Строки:** 14-30

**Проблема:**
```swift
func start() throws {
    let session = AVAudioSession.sharedInstance()
    try session.setCategory(.playback, mode: .default, options: [.mixWithOthers])
    // ...
}
```

iOS может убить приложение в фоне, если оно потребляет слишком много ресурсов. Нет мониторинга состояния фона.

**Решение:**
Добавить обработку `UIApplication.willResignActiveNotification` и снижение активности в фоне.

---

### 8. **Отсутствие retry логики при старте**

**Файл:** `LocalProxyController.swift`  
**Строки:** 88-120

**Проблема:**
```swift
do {
    // ...
    try OlcRTCEngine.start(profile: profile, ...)
} catch {
    // Сразу fail, нет retry
    status = .failed
}
```

Если старт не удался из-за временной проблемы (например, DNS timeout), приложение сразу падает в failed.

**Решение:**
Добавить 2-3 попытки с задержкой.

## 💡 Улучшения стабильности

### 9. **Улучшить обработку смены сети**

**Текущая логика:**
```swift
private func handlePathUpdate(_ snapshot: NetworkPathSnapshot) {
    // Сразу реагирует на изменение
    enterExternalTunnelRecovery("Network changed to \(snapshot.name)")
}
```

**Проблема:**
- Слишком агрессивная реакция
- Нет различия между Wi-Fi → LTE и Wi-Fi → другой Wi-Fi
- Нет проверки, действительно ли соединение сломалось

**Решение:**
1. Добавить debounce (2-3 секунды)
2. Сначала проверить connectivity, потом решать о рестарте
3. Различать типы переключений

## 🔧 Конкретные улучшения кода

### 12. **Оптимизировать SOCKS проверку**

```swift
for target in SocksConnectTarget.defaults {
```

**Решение:**

### 13. **Добавить graceful shutdown**

**Текущее:**
```swift
func stop() {
    stopEngineInBackground()
    // Сразу меняем статус
    status = .stopped
}
```

**Проблема:**
Не ждём завершения остановки engine.

**Решение:**
Дождаться завершения и только потом менять статус.

---

### 14. **Улучшить обработку портов**

**Текущее:**
```swift
let port = PortAvailability.nextAvailableTCPPort(startingAt: requestedPort)
```

**Проблема:**
Если порт занят, берём следующий. Но не сохраняем новый порт как предпочтительный.
