# 🔔 Гайд по уведомлениям

## Обзор

Система уведомлений OlcRTC Gateway информирует вас о важных событиях, даже когда приложение работает в фоне.

---

## 📱 Типы уведомлений

### 1. 🟢 Соединение восстановлено
**Когда:** SOCKS прокси снова заработал после сбоя  
**Сообщение:** "SOCKS прокси снова работает. Можно включить VPN."  
**Действие:** Включите профиль во внешнем VPN-клиенте

**Пример:**
```
┌─────────────────────────────────┐
│ 🟢 Соединение восстановлено     │
│ SOCKS прокси снова работает.    │
│ Можно включить VPN.             │
└─────────────────────────────────┘
```

---

### 2. 🔴 Ошибка подключения
**Когда:** Не удалось запустить SOCKS прокси  
**Сообщение:** Причина ошибки (например, "Tunnel connectivity failed")  
**Действие:** Проверьте профиль и попробуйте снова

**Пример:**
```
┌─────────────────────────────────┐
│ 🔴 Ошибка подключения           │
│ Не удалось запустить после      │
│ 3 попыток                       │
└─────────────────────────────────┘
```

---

### 3. 🟠 Сеть изменилась
**Когда:** Произошла смена сети (Wi-Fi ↔ LTE)  
**Сообщение:** "Перезапустите SOCKS и внешний VPN"  
**Действие:** 
1. Выключите туннель во внешнем VPN-клиенте
2. Нажмите "Перезапустить" в OlcRTC Gateway
3. Включите туннель обратно

**Пример:**
```
┌─────────────────────────────────┐
│ 🟠 Сеть изменилась              │
│ Перезапустите SOCKS и           │
│ внешний VPN                     │
└─────────────────────────────────┘
```

---

### 4. 🟠 Требуется действие
**Когда:** Нужно вмешательство пользователя  
**Сообщение:** Описание требуемого действия  
**Действие:** Откройте приложение и следуйте инструкциям

**Пример:**
```
┌─────────────────────────────────┐
│ 🟠 Требуется действие           │
│ Требуется перезапуск SOCKS      │
│ и внешнего VPN                  │
└─────────────────────────────────┘
```

---

### 5. ⭐ Долгая работа
**Когда:** Приложение работает без проблем 6, 12 или 24 часа  
**Сообщение:** "Приложение работает без проблем уже X часов!"  
**Действие:** Никакого, это поздравление! 🎉

**Пример:**
```
┌─────────────────────────────────┐
│ ⭐ Отличная работа!             │
│ Приложение работает без         │
│ проблем уже 12 часов!           │
└─────────────────────────────────┘
```

### Глобальный переключатель

**Включить уведомления** - главный переключатель:
- ✅ **Включено:** Все выбранные типы уведомлений будут отправляться
- ❌ **Выключено:** Уведомления не будут отправляться (даже если типы включены)

**Когда использовать:**
- Временно отключить все уведомления (например, на ночь)
- Быстро включить/выключить без изменения настроек типов

---

### Индивидуальные переключатели

Каждый тип уведомлений можно включить/выключить отдельно:

| 🔴 Ошибка подключения | ✅ Включить | Критическая информация |

| ⭐ Долгая работа | ⚠️ По желанию | Мотивационное, не критично |

## 🧪 Тестирование

### Отправка тестового уведомления

1. Откройте панель "Уведомления"
2. Нажмите кнопку **"Тест"**
3. Вы получите тестовое уведомление:

```
┌─────────────────────────────────┐

└─────────────────────────────────┘
```

**Примечание:** Если уведомление не пришло:
- Проверьте, что уведомления разрешены в iOS
- Проверьте, что глобальный переключатель включён
- Проверьте, что тип "Соединение восстановлено" включён

## 🔧 Устранение проблем

### Уведомления не приходят

**Проблема:** Уведомления не отображаются

**Решение:**

1. **Проверьте разрешения iOS:**
   - Откройте Настройки iOS
   - Найдите "OlcRTC Gateway"
   - Убедитесь, что "Уведомления" включены
   - Проверьте, что включены "Баннеры" и "Звуки"

2. **Проверьте настройки приложения:**
   - Откройте панель "Уведомления"
   - Убедитесь, что глобальный переключатель включён
   - Проверьте, что нужные типы включены

3. **Отправьте тестовое уведомление:**
   - Нажмите кнопку "Тест"
   - Если тест не пришёл - проблема в разрешениях iOS
   - Если тест пришёл - проблема в настройках типов

---

### Слишком много уведомлений

**Проблема:** Приходит слишком много уведомлений

**Решение:**

1. **Отключите ненужные типы:**
   - Откройте настройки уведомлений
   - Выключите типы, которые вам не нужны
   - Например, можно отключить "Долгая работа"

2. **Временно отключите все:**
   - Выключите глобальный переключатель
   - Настройки типов сохранятся
   - Включите обратно, когда нужно

---

### Уведомления приходят в неподходящее время

**Проблема:** Уведомления мешают (например, ночью)

**Решение:**

1. **Используйте режим "Не беспокоить" iOS:**
   - Настройте расписание в Настройках iOS
   - Уведомления будут приходить, но без звука

2. **Временно отключите уведомления:**
   - Выключите глобальный переключатель перед сном
   - Включите утром

3. **Настройте Focus Mode (iOS 15+):**
   - Создайте режим "Сон"
   - Отключите уведомления от OlcRTC Gateway
   - Режим будет включаться автоматически

## 🎯 Сценарии использования

### Сценарий 1: Смена сети

**Ситуация:** Вы вышли из дома (Wi-Fi → LTE)

**Что происходит:**
1. Приложение обнаруживает смену сети
2. Проверяет, работает ли туннель
3. Если туннель сломался, отправляет уведомление:

```
┌─────────────────────────────────┐
│ 🟠 Сеть изменилась              │
│ Перезапустите SOCKS и           │
│ внешний VPN                     │
└─────────────────────────────────┘
```

**Ваши действия:**
1. Откройте внешний VPN-клиент
2. Выключите туннель
3. Откройте OlcRTC Gateway
4. Нажмите "Перезапустить"
5. Вернитесь во внешний VPN-клиент
6. Включите туннель обратно

---

### Сценарий 2: Ошибка запуска

**Ситуация:** Не удалось запустить SOCKS прокси

**Что происходит:**
1. Приложение пытается запустить 3 раза
2. Все попытки неудачны
3. Отправляет уведомление:

```
┌─────────────────────────────────┐
│ 🔴 Ошибка подключения           │
│ Не удалось запустить после      │
│ 3 попыток                       │
└─────────────────────────────────┘
```

**Ваши действия:**
1. Откройте приложение
2. Проверьте логи в панели "Диагностика"
3. Проверьте профиль (возможно, истёк срок)
4. Попробуйте другой профиль
5. Если не помогает - обратитесь в поддержку

---

### Сценарий 3: Долгая работа

**Ситуация:** Приложение работает без проблем 12 часов

**Что происходит:**
1. Каждый час приложение проверяет uptime
2. При достижении 6/12/24 часов отправляет уведомление:

```
┌─────────────────────────────────┐
│ ⭐ Отличная работа!             │
│ Приложение работает без         │
│ проблем уже 12 часов!           │
└─────────────────────────────────┘
```

**Ваши действия:**
- Никаких! Это просто поздравление 🎉
- Можете отключить этот тип, если не нужен

## 🔐 Приватность

### Какие данные собираются?

**Никакие!** Уведомления:
- Отправляются локально на вашем устройстве
- Не передаются на сервер
- Не содержат личных данных
- Не отслеживаются

### Что хранится?

Только настройки уведомлений:
- Включены ли уведомления (да/нет)
- Какие типы включены (да/нет для каждого)
- Хранятся локально в UserDefaults
- Не синхронизируются с iCloud

## 🆘 Поддержка

### Часто задаваемые вопросы

**Q: Можно ли настроить звук уведомлений?**  
A: Да, в Настройках iOS → OlcRTC Gateway → Уведомления → Звуки

**Q: Можно ли изменить стиль баннера?**  
A: Да, в Настройках iOS → OlcRTC Gateway → Уведомления → Стиль баннера

**Q: Уведомления приходят, когда приложение открыто?**  
A: Да, они отображаются как баннер вверху экрана

**Q: Можно ли отключить badge (красный кружок)?**  
A: Да, в Настройках iOS → OlcRTC Gateway → Уведомления → Значки

**Q: Уведомления работают в режиме "Не беспокоить"?**  
A: Нет, они будут доставлены, но без звука и вибрации
